切法一
縱切：M1–M6 全做，每個輕量化
M1 到 M6 全部覆蓋，但每個模組只做「有問題 / 沒問題 + 差多少」的二元輸出，不給修改建議，不給連鎖影響。設計師從接案一路可以用到業主提報，但每個環節的深度都是最薄的版本。
✅ 優點
  • 可驗證完整工作流是否跑得通，從 M1 到 M6 都有產出
  • 能觸發跨模組的連鎖痛點，例如 M3 骨架改動影響 M4 驗算
  • 設計師看到的是完整流程，比較容易評估整體價值
  • 開發分工清楚，各模組可以小組並行
⚠️ 缺點
  • 每個功能都粗糙，沒有任何一個做到足以替代現有工作方式
  • 設計師一直看到警告卻不知道怎麼辦，挫折感可能高於沒有工具
  • 資源分散，很難在任何一個模組建立深度競爭壁壘
  • 付費意願難以驗證，因為每個功能都不完整
最大風險：「只警告不建議」的體驗。設計師知道 53m 超標了，但系統不告訴他梯廳要往哪移。這個體驗可能讓設計師覺得系統只是多了一個需要關注的警示視窗，而不是真正的助手。

開發路徑
三階段 Roadmap
Phase 1
3–4 個月
M1 + M2 + M3 輕量版上線
建立從法規查找、基地分析到骨架配置的輕量流程，每個環節只做二元合規輸出。
M1：適用法規清單輸出，邊界條件手動填
M2：退縮線 overlay 顯示
M3：50m 步行距離邊界圓標示
驗證：設計師是否願意在案件起點就進入系統，還是覺得太早進來沒有價值？
Phase 2
3–4 個月
M4 + M5 + M6 輕量版補足
補足標準層配置階段的合規輸出，讓完整工作流跑通。
M4：步行距離、日照合規 / 不合規 + 超標量顯示
M5：修改受影響元素清單（不給建議）
M6：靜態面積表輸出
驗證：完整流程跑通後，設計師整體付費意願是否有提升？哪個模組最常被使用？
Phase 3
4–6 個月
依使用率決定哪個模組先升級為完整版
依 Phase 1–2 的使用資料，決定優先把哪個模組從「輕量版」升級為「給建議版」。
最高使用率的模組優先升級，加入修改建議與連鎖影響
逐步往 M3–M5 顧問閉環路徑靠攏
切法二
橫切：M1、M2 先做深，再逐步串接
挑出獨立性最高的模組——M1 法規分析、M2 基地分析——先做到足夠深度，讓設計師在這兩個環節真的用得起來並願意付費。確認商業價值後，再逐步往後串接。
✅ 優點
  • 失敗成本低：M1/M2 做不起來，損失僅限於這兩個模組
  • 可多團隊並行：M1 和 M2 互不依賴，可同步開發
  • 可先驗證「前期資料收斂」的付費意願，再決定是否往後延伸
  • M1 每案都必須執行，是最穩定的高頻入口
⚠️ 缺點
  • M5 連鎖影響（最大痛點）根本不會出現在 MVP 裡
  • M1/M2 的「顧問性質」較弱，偏資料整理而非設計判斷
  • 驗到的是「前期工具」的付費意願，不是「設計顧問」的付費意願
  • 後期串接 M3–M5 需要重新設計資料模型，可能有技術債
定位風險：M1/M2 做深之後，市場定位容易滑向「法規查找工具」而不是「設計輔助顧問」。這兩個定位的付費意願、客群、競爭對手都不同，需要在 Phase 1 就明確選邊。

切法三
M3–M5 顧問閉環：聚焦設計輔助顧問定位
聚焦 M3（骨架）→ M4（標準層配置同步驗算）→ M5（連鎖影響橫切服務）這條核心鏈。系統在設計師配置標準層時，同步偵測合規問題、給出具體繪圖修改建議，並標示修改的連鎖影響範圍。M3/M4/M5 缺一不可，這三個合在一起才是完整的設計顧問閉環。
✅ 優點
  • 直接驗證最高付費意願的定位：設計師「修改 + 連鎖影響判斷」最貴最難
  • 每次設計修改都重新觸發，付費意願是持續性的而不是一次性的
  • M5 連鎖影響是最難複製的護城河，從一開始就建立壁壘
  • 定位清晰：設計輔助顧問，而非法規查找工具或計算工具
⚠️ 缺點
  • 需要 CAD 匯入作為前提，技術門檻比前兩種高
  • M3/M4/M5 強依賴，任何一個做不好整個閉環就不完整
  • 修改建議的「可信度」是關鍵挑戰——設計師是否真的採納？
  • 前期需要較多設計師配合（標記梯廳、戶型邊界等）
前提條件：CAD 匯入策略決定這條路的可行性。MVP 採用「手動標記關鍵點」（DWG 作為底圖，設計師點選梯廳、戶型邊界、外框線），技術門檻最低，但每次版本更新需重新標記是主要摩擦點。

Phase 1

M1：適用法規自動彙整，結果直接作為 M3 的合規依據
M2：可建範圍自動產出，直接作為 M3 骨架的空間約束
CAD 圖層標記輔助（方案②）：約定命名規範 + CAD 外掛
驗證指標：同一案子跨多個設計迴圈的留存率；M1 法規收斂是否確實減少 M3 的錯誤發生率。
Phase 3

確認清單
與建築師討論的關鍵問題
這四個問題的答案，會直接決定三種切法中哪一條最適合現在的階段。
確認問題 01
縱切版「只警告不建議」的使用體驗
設計師一直看到警告卻不知道怎麼辦，是否比沒有工具更令人沮喪？可接受度是否足夠支撐付費？
確認問題 02
M3–M5 顧問閉環的 CAD 標記摩擦
每次設計版本更新後手動重新標記梯廳、戶型邊界，設計師的操作成本是否可接受？
確認問題 03
M5 連鎖影響的粒度
「受影響元素清單」要細到什麼程度才有用？列出「戶型 B3 需重畫」夠用，還是需要說明怎麼改？
確認問題 04
付費模式偏好
按案收費、月訂閱、還是模組分開定價？設計師習慣在哪個決策點付費？
